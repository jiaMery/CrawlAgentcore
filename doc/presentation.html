背景

传统爬虫的痛点

❌ 每个目标网站需要手写定制爬虫脚本
❌ 反爬虫策略需要人工调整并发和延迟
❌ JS 渲染页面需要额外部署 Selenium/Playwright
❌ 中文/CJK 字符编码损坏问题
❌ 无可观测性，调试困难
✅ 自然语言描述 → 自动生成并执行爬虫代码
✅ 按域名自动分级，智能调整并发策略
✅ 托管 Chromium，真实 JS 渲染 + 反爬绕过
✅ 多层编码管道，完整 CJK/Unicode 支持
✅ OpenTelemetry → X-Ray → CloudWatch 全链路追踪

编码管道

CJK 编码管道 — 零数据损失

AgentCore 传输层会损坏 0x80–0x9F 非 ASCII 字节，以下管道保证中文完整传输

// Code Interpreter 标准输出
                     ↓
_safe_output(result)   // SKILL.md 强制要求 LLM 调用
    json.dumps → base64.b64encode → print <<>>
                     ↓
_extract_crawler_output(agent)
    ① ast.literal_eval 解包 CI 的 [{'type':'text','text':'...'}] 格式
    ② regex 匹配 <<>> → base64.b64decode → json.loads
    ③ 降级：<<>> + Mojibake 修复
    ④ 降级：裸 JSON 启发式解析
                     ↓
_ensure_ascii_safe()  // 转义所有非 ASCII → \uXXXX
                     ↓ AgentCore 传输
_unescape_unicode_recursive() // CLI 端还原 \uXXXX → 真实 Unicode
Mojibake 修复： _repair_mojibake() 检测 UTF-8 被误以 Latin-1 解码（最多 3 次往返修复）；_looks_like_mojibake() 触发自动重试爬取

关键问题解决：
容器内短期 IAM Task Role Token 会在运行期间过期，导致 OTLP 400 错误。
_RefreshingAWS4Auth 实现 __call__ 接口，每次 HTTP 请求时调用 get_frozen_credentials() 重新获取凭证。
快速开始

本地开发 & 云端调用

本地开发模式

# 1. 创建虚拟环境（需要 /usr/bin/python3.10）
/usr/bin/python3.10 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. 启动开发服务器
AWS_REGION=us-east-1 \
  .venv/bin/uvicorn src.main:app \
  --host 0.0.0.0 --port 8080

# 3. 本地爬取
python crawler_cli.py --dev \
  "爬取 https://books.toscrape.com 所有书价"

云端调用示例

# Code Interpreter 模式（默认）
python crawler_cli.py --cloud \
  "爬取豆瓣电影TOP250的名称、评分、导演"

# Browser 模式（JS 渲染 / 反爬）
python crawler_cli.py --cloud --browser \
  "抓取某动态加载商品页面"

# 指定技能 + 保存结果
python crawler_cli.py --cloud \
  --skill news-crawl \
  --output result.json \
  "https://news.ycombinator.com"

# 一键部署所有 AWS 资源
./deploy/deploy.sh crawl-agentcore \
  us-east-1 my-s3-bucket
