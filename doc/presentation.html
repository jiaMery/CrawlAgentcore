AgentCore 组件

AgentCore 核心组件全览

⚡ AgentCore Runtime
托管 Agent 容器运行时，支持 HTTP 协议，弹性扩展；I/O 等待期间不计费；按实际 CPU/内存秒级计费
🐍 Code Interpreter
隔离沙箱执行 LLM 生成的 Python/JS 代码；内置 requests、pandas、numpy 等常用库；每次执行后自动清理
🌐 AgentCore Browser
托管 Chromium 实例，支持 CDP WebSocket 自动化；JS 渲染、截图、表单操作；按使用时长计费
🧠 Memory
跨会话持久化 Agent 记忆；支持短期（对话内）和长期（跨对话）两种模式；自动语义检索
🔌 Gateway（MCP 网关）
将内部 API、Lambda 函数、知识库统一暴露为 MCP Server；Agent 通过标准协议调用任意工具
🔐 Identity & Auth
原生 IAM 集成；自动化推理验证访问策略（与 S3/IAM 同底层技术）；支持 OAuth / API Key
🔭 Observability（可观测性）
OTLP → AWS X-Ray → CloudWatch；追踪 Agent 每步操作和工具调用；支持 A/B 测试
🔒 企业级安全：所有组件均与 IAM 深度集成，支持 VPC 私网部署，多区域高可用，内置合规支持（SOC、ISO、HIPAA）

传统爬虫的痛点

自建方案在规模化落地时面临的系统性挑战
传统痛点

为什么传统爬虫难以规模化？

🔧 开发成本高

每个目标站点需定制爬虫代码；网站改版后需重写选择器；技术门槛高，依赖专职工程师维护

🛡️ 反爬对抗复杂

需人工调整 User-Agent、IP 轮换、请求频率；遭遇 CAPTCHA、JS 指纹检测时束手无策；维护成本随站点数量线性增长

🌐 JS 渲染页面困难

React/Vue SPA 用 requests 只能得到空白 HTML；需自建 Selenium/Playwright 集群，资源消耗大

⚙️ 运维负担重

需自建调度系统、监控告警、失败重试；EC2/容器集群需 7×24 维护；扩容缩容需手动操作

🀄 编码问题频发

中文/日文等 CJK 字符在传输过程中乱码；charset 自动检测不准确导致数据损坏

📊 可观测性缺失

无法追踪爬取任务耗时分布；异常定位困难，依赖日志人工排查
* 以上痛点来源于典型企业自建爬虫项目的真实运营反馈

AgentCore 优势

AWS AgentCore 的核心差异化优势

🤖 AI 原生代码生成

基于 Claude Sonnet，根据自然语言指令动态生成爬虫脚本；网站改版时只需重新描述需求，无需修改代码

🔒 隔离沙箱执行

Code Interpreter 在独立沙箱中运行生成的爬虫代码，安全隔离，不影响主应用；每次爬取后自动清理

🌐 真实浏览器托管

AgentCore Browser 提供托管 Chromium + CDP WebSocket；支持 JS 渲染、截图、Canvas 指纹绕过，无需运维浏览器集群

⚡ 按实际 CPU/内存计费

I/O 等待期间（占爬取时间 30–70%）不计费；相比 EC2 按小时付费可节省大量成本

🔐 原生 IAM 集成

容器自动获得 Task Role 凭证；无需管理密钥，通过 IAM Policy 精细控制权限

📊 全链路可观测性

Strands Agents 自动插桩；OTLP → X-Ray → CloudWatch，每个 Agent 调用步骤均可追踪，无需额外埋点
